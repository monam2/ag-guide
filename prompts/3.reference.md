## Few-Shot 프롬프팅

AI에게 몇 가지 예시를 제공하는 방법. 아무런 예시없이 AI에게 프롬프팅하면 AI의 창의성과 자유도에 너무 많은 기대를 걸게 된다.

```
다음 텍스트의 감정을 분석해줘: "이 제품 정말 별로네요."
-> 몇 개의 예제를 함께 제공해주면 출력 형태가 명확해지고 예측 가능해진다.

다음 텍스트의 감정을 분석해줘.
예시:
- "정말 좋아요 추천합니다" -> 긍정
- "그냥 그래요" -> 중립
- "다시는 안 살래요" -> 부정
```

-> 규격/규칙에 딱 들어맞는 구현을 할 때 Few-Shot 프롬프팅을 사용하면 효율적이다.

```
다음 패턴으로 API핸들러를 만들어줘.
[예시: 사용자 조회 API]

export async function GET(
    request: Request,

```

-> 예시 사용이 가장 빛나는 경우: 포맷 변환

## 효과적인 예시 제공하기

- 대표성 : 생성하려는 코드와 유사한 구조의 예시 선택
- 완전성 : 예시는 완전해야 한다. 일부만 보여주면 AI가 나머지를 추측하므로 좋지 않다.
- 복잡도 : 단순한 코드를 만들 때 복잡한 예시를 주지 마라. 예시가 복잡하면 핵심 패턴이 묻힌다.
- 품질 : 예시 코드 자체가 좋은 코드여야 한다. AI는 예시의 나쁜 패턴도 따라한다.
