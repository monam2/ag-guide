# Evaluate & Iterate

## 평가 기준 설정

좋은 프롬프트와 나쁜 프롬프트를 가르는 기준이 필요하고, 프롬프로 생성한 대답을 평가해 이를 나눌 수 있다.

- 기능적 정확성
  - 요구사항을 정확히 구현했는가?
  - 에러 케이스를 적절히 처리했는가?
  - 엣지 케이스를 고려했는가?
  - 보안 취약점은 없는가?
  - 성능 요구사항을 만족하는가?
  - 버그가 없는가?
- 코드 품질
  - 가독성이 좋은가?
  - 기존 코드와 스타일이 일관되는가?
  - 불필요한 복잡성이 없는가?
  - 재사용이 용이한가?
  - 디버깅하기에 적절한가?
- 성능
  - 효율적인 알고리즘을 사용하는가?
  - 불필요한 연산/계산이 없는가?
  - 메모리를 효율적으로 사용하는가?
- 보안
  - 입력 검증이 적절한가?
  - 민감 정보가 노출되지 않는가?
  - 인증/인가가 적절히 구현되었는가?
  - 보안 취약점은 없는가?
  - SQL 인젝션, XSS 등 방어책이 있는가?
- 유지보수성
  - 테스트가 용이한가?
  - 확장 가능한가?
  - 문서화가 적절한가?

## 반복 개선의 중요성

```
이거 별로야. 다시 말해줘
->
다음 문제를 수정해줘:
1. validatePassword 함수에서 특수문자 검증이 누락됨
- 현재: 영문 + 숫자만 검증
- 필요: !@#$%^&*()중 하나 이상 포함 필수
2. 오류 메시지가 영어로 되어 있음
- "Password must be at least 8 characters long" -> "비밀번호는 최소 8자 이상이어야 합니다."
- "Password must contain at least one special character" -> "비밀번호는 특수문자를 하나 이상 포함해야 합니다."
3. 반환 타입에 오류 코드가 없음
현재: {isValid: false, message: string}
필요: {isValid: false, message: string, code: 'WEAK_PASSWORD' | 'INVALID_PASSWORD' | 'UNKNOWN_ERROR'}
```
